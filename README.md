# He Gu

I am working on cognition infrastructure for long-horizon human-AI collaboration.

The current thesis is simple:

> The next bottleneck in AI agents is not only capability.
> It is preserving human judgment while the agent, memory, and workflow keep evolving.

## Current Work

### [coevolution-kernel](https://github.com/hegu-1/coevolution-kernel)

A position paper on the missing kernel layer between human judgment and self-evolving AI agents: provenance, drift protection, co-evolution, and judgment integrity.

It connects three failure modes that are usually discussed separately:

- self-evolving agents drifting silently
- long-term memory accumulating without provenance
- AI translation flattening human judgment into generic language

### [claude-cognition-skills](https://github.com/hegu-1/claude-cognition-skills)

Six Claude Code skills for the failure modes of real long-horizon collaboration:

- continuity loss
- premature closure
- abstraction drift
- output before structure
- decision overload
- faithful translation without erasing the original judgment

### [personal-ai-os](https://github.com/hegu-1/personal-ai-os)

An early Personal AI OS prototype exploring continuity snapshots, agent handoffs, alignment contracts, and user-owned cognitive state.

## Open Questions

I am especially interested in:

- agent memory provenance
- runtime drift detection
- long-horizon Claude Code / coding-agent workflows
- faithful translation of nonlinear human thought
- user-level alignment layers outside model training
- personal AI systems where the human remains central, evolving, and auditable

If you are working on similar problems, open an issue in one of the repos or reach out.
